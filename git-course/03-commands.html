Section 3

Essential Commands

The 20 commands you will use 90% of the time. Each includes real-world context and common flags.

💡

Run git <command> --help at any time to open the full manual page for any command. For a quick summary: git <command> -h.

Setup & Initialization

git init

Creates a new repository in the current directory (adds a .git/ folder).

git init
git init my-project    # creates a subdirectory

git clone

Downloads a full copy of a remote repository, including all history.

# Clone from GitHub (HTTPS)
git clone https://github.com/org/repo.git

# Clone to a specific folder name
git clone https://github.com/org/repo.git my-local-name

# Shallow clone — only the last N commits (faster for large repos)
git clone --depth 1 https://github.com/org/repo.git

# Clone a specific branch
git clone -b release/2.4 https://github.com/org/repo.git
🔍

TSE use: You will clone repos constantly — to reproduce bugs, inspect configs, or run tests at a specific version. Prefer shallow clones (--depth 1) on large repos when you only need the current state.

Inspecting State

git status

Shows the state of your working directory and staging area. Your first command after any change.

git status
On branch main
Your branch is up to date with 'origin/main'.

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   app.py

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
        modified:   utils.py

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_feature.py

git status -s   # short format
M  app.py
 M utils.py
?? new_feature.py

git log

Shows commit history. This is one of the most powerful investigation tools.

# Standard log
git log

# Compact one-line per commit
git log --oneline
f1a2b3 (HEAD -> main) fix: null pointer in login flow
789abc feat: add search endpoint
d4e5f6 feat: user auth

# Graph view — shows branches and merges visually
git log --oneline --graph --all
* f1a2b3 (main) hotfix
| * cc1122 (feature/search) feat: search UI
| * dd3344 feat: search API
|/
* 789abc fix crash

# Filter: commits by a specific author
git log --author="Alice"

# Filter: commits touching a specific file
git log --follow -- path/to/file.py

# Filter: commits between two tags
git log v2.3.0..v2.4.0 --oneline

# Show what changed in each commit (patch format)
git log -p

# Show stats (lines changed per file)
git log --stat

git diff

Shows the actual changes between states.

# Working directory vs staging area (unstaged changes)
git diff

# Staging area vs last commit (what's about to be committed)
git diff --staged

# Between two commits
git diff abc1234 def5678

# Between two branches
git diff main..feature/search

# Only show changed file names (not the actual diff)
git diff --name-only v2.3.0..v2.4.0

git show

Shows details of a specific commit — author, date, message, and diff.

git show f1a2b3
git show HEAD            # current commit
git show HEAD~2           # 2 commits back
git show v2.4.0          # a tag
git show HEAD:path/to/file.py  # view a file at HEAD

git blame

Shows who last changed each line of a file and in which commit.

git blame app.py
^a1b2c3 (Alice 2024-01-10) def authenticate(user):
 d4e5f6 (Bob   2024-02-15)     token = generate_token(user)
 789abc (Carol 2024-03-01)     if token is None:

# Show only lines 20-40
git blame -L 20,40 app.py

# Ignore whitespace changes
git blame -w app.py

Staging & Committing

git add

# Stage a specific file
git add app.py

# Stage all changes in current directory
git add .

# Stage parts of a file interactively (hunk-by-hunk)
git add -p app.py

git commit

# Opens your configured editor for the message
git commit

# Inline message
git commit -m "fix: handle null token in auth flow"

# Stage all tracked files AND commit in one step
git commit -am "fix: handle null token"

# Amend the last commit (message or content) — only before pushing!
git commit --amend
💡

Conventional Commits: Many teams use type: description format — e.g. fix: null pointer, feat: add login, docs: update README, chore: bump deps. This powers automated changelogs and semantic versioning.

Branching

git branch / git checkout / git switch

# List local branches (* = current)
git branch

# Create a new branch
git branch feature/my-feature

# Switch to a branch
git switch feature/my-feature   # modern syntax
git checkout feature/my-feature  # classic syntax

# Create AND switch in one step
git switch -c feature/new-branch
git checkout -b feature/new-branch  # classic

# Delete a merged branch
git branch -d feature/done

# Force-delete an unmerged branch
git branch -D feature/abandoned

Working with Remotes

git remote

# List remotes
git remote -v
origin  https://github.com/org/repo.git (fetch)
origin  https://github.com/org/repo.git (push)

# Add a remote (useful when working with forks)
git remote add upstream https://github.com/original-org/repo.git

git fetch / git pull / git push

# Fetch: download remote changes without merging
git fetch origin

# Pull: fetch + merge (or rebase with --rebase)
git pull origin main
git pull --rebase origin main

# Push your local branch to the remote
git push origin feature/my-feature

# Set upstream and push in one step (first push only)
git push -u origin feature/my-feature

# Delete a remote branch
git push origin --delete feature/old-branch

Undoing Things

⚠️

Some undo operations are destructive. When in doubt, create a backup branch first: git branch backup-$(date +%s)

git restore

# Discard changes in working directory (irreversible!)
git restore app.py

# Unstage a file (keep working-dir changes)
git restore --staged app.py

# Restore a file to its state at a specific commit
git restore --source abc1234 app.py

git revert

Creates a new commit that undoes a previous commit. Safe to use on shared branches.

# Revert a specific commit by adding an inverse commit
git revert abc1234

# Revert without immediately committing (let you edit the message)
git revert --no-commit abc1234

git reset

Moves the branch pointer backward. Modes control what happens to your changes:

ModeStaging areaWorking dirUse case
--soft Keeps changes staged Unchanged Undo commit, re-edit message
--mixed (default) Unstages changes Unchanged Undo commit, re-stage selectively
--hard Discards changes Discards changes Completely abandon commits (destructive) 
# Undo last commit, keep changes staged
git reset --soft HEAD~1

# Undo last commit, keep changes unstaged
git reset HEAD~1

# ⚠ Destroy last 2 commits completely
git reset --hard HEAD~2

git stash

Temporarily saves dirty working-directory changes without committing — useful when you need to switch context quickly.

# Save current changes
git stash
git stash push -m "WIP: partial search feature"

# List stashes
git stash list
stash@{0}: WIP: partial search feature
stash@{1}: On main: experiment

# Apply the most recent stash (keeps it in list)
git stash apply

# Apply and remove from list
git stash pop

# Apply a specific stash
git stash apply stash@{1}
