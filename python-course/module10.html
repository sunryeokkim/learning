Two Sum
Easy Hash Map

Problem: Given an array nums and a target integer, return indices of the two numbers that add up to target. Each input has exactly one solution.

Example: nums=[2,7,11,15], target=9 → [0,1] (2+7=9)

Thought Process
Brute force: Try every pair (i,j) where i < j. Check if nums[i]+nums[j]==target. O(n²) time.

Optimization: For each number x, we need to find target-x. If we've already seen target-x, we have our answer. Store seen values and their indices in a hash map as we scan. This gives O(n) time. 
def two_sum(nums, target):
    seen = {}                       # value → index
    for i, x in enumerate(nums):
        complement = target - x
        if complement in seen:      # O(1) lookup
            return [seen[complement], i]
        seen[x] = i
    return []

# Time: O(n) | Space: O(n)

Longest Substring Without Repeating Characters
Medium Sliding Window

Problem: Given string s, find the length of the longest substring with no repeating characters.

Example: s="abcabcbb" → 3 ("abc")

Thought Process
Key insight: The answer is some contiguous window of characters. We want the longest window where every character appears at most once.

Pattern: Variable sliding window. Expand right, shrink left when we see a duplicate. Track the last-seen index of each character to jump the left pointer past duplicates in O(1) instead of crawling. 
def length_of_longest_substring(s):
    last_seen = {}   # char → last index where we saw it
    left = best = 0

    for right, c in enumerate(s):
        # If c was seen within our current window [left, right], shrink
        if c in last_seen and last_seen[c] >= left:
            left = last_seen[c] + 1   # jump left past the previous occurrence

        last_seen[c] = right
        best = max(best, right - left + 1)

    return best

# Time: O(n) | Space: O(min(n, alphabet_size))

Product of Array Except Self
Medium Prefix / Suffix Product

Problem: Given array nums, return an array where output[i] is the product of all elements except nums[i]. No division allowed. O(n) time, O(1) extra space.

Example: [1,2,3,4] → [24,12,8,6]

Thought Process
If division allowed: total_product / nums[i]. But division is banned and nums might have zeros.

Key insight: output[i] = (product of all elements LEFT of i) × (product of all elements RIGHT of i).

Two passes: pass 1 fills left products, pass 2 multiplies in right products on-the-fly.
Two-pass trace for [1, 2, 3, 4]
idx: 0 1 2 3 nums: [1, 2, 3, 4] Pass 1 (left products): result = [1, 1, 2, 6] (result[i] = product of nums[0..i-1]) Pass 2 (multiply in right products, right-to-left): right = 1 i=3: result[3] = 6*1 = 6, right = 1*4 = 4 i=2: result[2] = 2*4 = 8, right = 4*3 = 12 i=1: result[1] = 1*12 = 12, right = 12*2 = 24 i=0: result[0] = 1*24 = 24, right = 24*1 = 24 Final: [24, 12, 8, 6] ✓ 
def product_except_self(nums):
    n = len(nums)
    result = [1] * n

    # Pass 1: result[i] = product of all elements to the LEFT of i
    left_product = 1
    for i in range(n):
        result[i] = left_product
        left_product *= nums[i]

    # Pass 2: multiply in product of all elements to the RIGHT of i
    right_product = 1
    for i in range(n - 1, -1, -1):
        result[i] *= right_product
        right_product *= nums[i]

    return result

# Time: O(n) | Space: O(1) (result array doesn't count)

Maximum Subarray (Kadane's Algorithm)
Medium Greedy / DP

Problem: Find the contiguous subarray with the largest sum.

Example: [-2,1,-3,4,-1,2,1,-5,4] → 6 (subarray [4,-1,2,1])

Thought Process
Key decision at each element: should we extend the previous subarray or start fresh?
Extend if current_sum + nums[i] > nums[i], which simplifies to current_sum > 0.

If the running sum ever goes negative, it only hurts us — discard it and start fresh. 
def max_subarray(nums):
    current = best = nums[0]
    for x in nums[1:]:
        # Either extend previous subarray or start new one at x
        current = max(x, current + x)
        best = max(best, current)
    return best

# DP perspective: dp[i] = max sum ending at index i
# dp[i] = max(nums[i], dp[i-1] + nums[i])
# Time: O(n) | Space: O(1)

Number of Islands
Medium DFS / BFS / Union-Find

Problem: Given a 2D grid of '1' (land) and '0' (water), count the number of islands. An island is surrounded by water and formed by connecting adjacent lands horizontally/vertically.

Thought Process
This is exactly "count connected components" on a grid graph. For each unvisited land cell, do a DFS/BFS to mark the entire connected component as visited. Count how many times we start a new DFS. 
def num_islands(grid):
    if not grid: return 0
    rows, cols = len(grid), len(grid[0])

    def dfs(r, c):
        # Base cases: out of bounds or water or already visited
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != '1':
            return
        grid[r][c] = '0'    # mark as visited (sink the island)
        dfs(r+1, c); dfs(r-1, c)
        dfs(r, c+1); dfs(r, c-1)

    islands = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                dfs(r, c)       # DFS marks entire island as visited
                islands += 1

    return islands

# Time: O(rows * cols) | Space: O(rows * cols) for recursion stack

Word Break
Medium DP / Memoized DFS

Problem: Given string s and a dictionary wordDict, determine if s can be segmented into a sequence of dictionary words.

Example: s="leetcode", wordDict=["leet","code"] → True

Thought Process
State: Can we segment s[i:]?
Transition: For each word in the dictionary, if s[i:i+len(word)] == word, then check if s[i+len(word):] can be segmented.
Base case: Empty string → True. With memoization, each index is computed once. 
def word_break(s, wordDict):
    word_set = set(wordDict)    # O(1) lookup
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True                # empty string can always be segmented

    for i in range(1, n + 1):
        for j in range(i):
            if dp[j] and s[j:i] in word_set:   # s[0..j-1] ok, and s[j..i-1] is a word
                dp[i] = True
                break           # found a valid split — no need to keep looking

    return dp[n]

# Alternatively, with @cache (top-down):
from functools import cache

def word_break_memo(s, wordDict):
    word_set = set(wordDict)
    @cache
    def can_break(i):
        if i == len(s): return True
        for word in word_set:
            if s[i:i+len(word)] == word and can_break(i + len(word)):
                return True
        return False
    return can_break(0)

# Time: O(n² * max_word_len) | Space: O(n)

Merge K Sorted Lists
Hard Heap

Problem: Merge k sorted linked lists and return the sorted list.

Thought Process
Naive: Merge lists two at a time → O(kN) total where N = total nodes.

Heap approach: The next element to add to the merged list is always the smallest among the current heads of all k lists. Use a min-heap to find this in O(log k). Total: O(N log k).

Python trap: Heap compares nodes, but ListNode has no comparison operator. Use (val, tie_breaker, node) to avoid comparisons between ListNode objects. 
import heapq

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def merge_k_lists(lists):
    heap = []
    counter = 0   # tie-breaker — prevents comparing ListNode objects

    for node in lists:
        if node:
            heapq.heappush(heap, (node.val, counter, node))
            counter += 1

    dummy = curr = ListNode(0)
    while heap:
        val, _, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, counter, node.next))
            counter += 1

    return dummy.next

# Time: O(N log k) | Space: O(k) — heap holds at most k elements

Course Schedule II (Topological Sort)
Medium Topological Sort / Kahn's

Problem: There are n courses (0 to n-1). prerequisites[i] = [a,b] means you must take course b before a. Return a valid ordering, or empty list if impossible.

Thought Process
This is topological sort on a directed graph where edges point from prerequisite to dependent. If there's a cycle, no ordering exists → return []. Use Kahn's algorithm: process nodes with in-degree 0 first, decrement neighbors' in-degrees, add them when they reach 0. 
from collections import defaultdict, deque

def find_order(n, prerequisites):
    graph = defaultdict(list)
    indegree = [0] * n

    for course, prereq in prerequisites:
        graph[prereq].append(course)
        indegree[course] += 1

    # Start with courses that have no prerequisites
    queue = deque([i for i in range(n) if indegree[i] == 0])
    order = []

    while queue:
        course = queue.popleft()
        order.append(course)
        for dependent in graph[course]:
            indegree[dependent] -= 1
            if indegree[dependent] == 0:
                queue.append(dependent)

    # If we processed all courses, a valid order exists; otherwise, cycle
    return order if len(order) == n else []

# Time: O(V + E) | Space: O(V + E)

Trapping Rain Water
Hard Two Pointers / Prefix Max

Problem: Given an array of bar heights, compute how much water can be trapped between bars after raining.

Example: [0,1,0,2,1,0,1,3,2,1,2,1] → 6

Thought Process
Key insight: Water at position i = min(max_height_left[i], max_height_right[i]) - height[i].

Two-pointer optimization: We don't need to precompute both prefix arrays. Maintain a left_max and right_max. The side with the smaller max is the bottleneck — process it and advance that pointer.
Why two pointers work
If left_max < right_max: water[left] = left_max - height[left] (guaranteed ≥ 0) We know right_max exists ≥ left_max, so left_max is the constraint. We can safely compute water for left and advance left pointer. Symmetric for the right side. 
def trap(height):
    if not height: return 0
    l, r = 0, len(height) - 1
    left_max = right_max = 0
    water = 0

    while l < r:
        if height[l] <= height[r]:          # left side is the bottleneck
            if height[l] >= left_max:
                left_max = height[l]         # update max, no water here
            else:
                water += left_max - height[l]  # water = max - current height
            l += 1
        else:                               # right side is the bottleneck
            if height[r] >= right_max:
                right_max = height[r]
            else:
                water += right_max - height[r]
            r -= 1

    return water

# Time: O(n) | Space: O(1)

Longest Palindromic Substring
Medium Expand Around Center

Problem: Find the longest palindromic substring in s.

Example: s="babad" → "bab" or "aba"

Thought Process
Brute force: Check all O(n²) substrings → O(n³).

DP approach: dp[i][j] = True if s[i..j] is palindrome. O(n²) time and space.

Expand around center: Every palindrome has a center (single char or two equal chars). Expand outward while chars match. O(n²) time, O(1) space — preferred. 
def longest_palindrome(s):
    start = length = 0

    def expand(l, r):
        nonlocal start, length
        while l >= 0 and r < len(s) and s[l] == s[r]:
            if r - l + 1 > length:
                start = l
                length = r - l + 1
            l -= 1
            r += 1

    for i in range(len(s)):
        expand(i, i)       # odd-length palindromes (center at i)
        expand(i, i + 1)   # even-length palindromes (center between i and i+1)

    return s[start:start + length]

# Time: O(n²) | Space: O(1)

# ── Pattern Recognition Summary ──────────────────────────────
# Here's the decision tree for choosing an approach:
#
# Is the array sorted?
#   → Binary search or two pointers
# Is the problem about "how many ways" or "minimum/maximum"?
#   → Dynamic programming
# Is it about connected components or shortest path?
#   → Graph algorithms (BFS/DFS/Dijkstra)
# Is it about contiguous subarrays with a constraint?
#   → Sliding window or prefix sums
# Is it about finding all possible combinations/arrangements?
#   → Backtracking
# Is the data hierarchical (nested dependencies, recursive structure)?
#   → Trees / recursion
