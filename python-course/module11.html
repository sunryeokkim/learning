Arithmetic Operators — Every Detail

Division: Three Operators You Must Know

Python has three separate division operators. This is different from C++/Java which have only one. Getting these wrong is a very common source of bugs.

# /   → True division — ALWAYS returns float
10 / 3      # 3.3333...
10 / 2      # 5.0       ← still a float, even when exact!
9 / 3       # 3.0

# //  → Floor division — rounds toward negative infinity
# WHY floor (not truncation): Python chose mathematical consistency over C-compat
10 // 3     # 3         ← floor(3.33) = 3
-10 // 3    # -4        ← floor(-3.33) = -4 (NOT -3!)
10 // -3    # -4        ← floor(-3.33) = -4
7 // 2      # 3
-7 // 2     # -4        ← TRAP: truncation would give -3, floor gives -4

# %   → Modulo — result has same sign as DIVISOR (in Python)
# WHY: Python guarantees a = (a//b)*b + (a%b); floor division forces this sign rule
10 % 3      # 1
-10 % 3     # 2         ← TRAP: not -1! Python: result is always >= 0 when divisor > 0
10 % -3     # -2        ← negative when divisor is negative
-7 % 2      # 1         ← always non-negative when divisor is positive

# Identity: a == (a // b) * b + (a % b)  — always holds in Python
# Check: -10 == (-4)*3 + 2 = -12 + 2 = -10 ✓

# divmod — both at once in one call (more efficient than doing both)
# WHY use divmod: single division instruction internally vs two separate ops
divmod(10, 3)    # (3, 1)   — (quotient, remainder)
divmod(-10, 3)   # (-4, 2)  — consistent with // and %
divmod(17, 5)    # (3, 2)

# ** → Exponentiation
2 ** 10         # 1024
(-2) ** 3       # -8
2 ** -1         # 0.5   ← WHY float: negative exponent can't be represented as int
2 ** 0          # 1
⚠️
Floor division vs truncation — the negative number trap

C++/Java truncate toward zero: -7/2 = -3. Python floors toward -∞: -7//2 = -4. If you're porting an algorithm, this is the most common source of off-by-one errors. When you need truncation: int(-7/2) gives -3.

Bitwise Operators

Bitwise operators work directly on the binary representation of integers. Essential for bitmask DP, flag manipulation, and interview bit tricks.

Binary representation review
Decimal 13 = binary 1101 bit 3 bit 2 bit 1 bit 0 1 1 0 1 Bit positions are 0-indexed from the RIGHT. Bit 0 (rightmost) = least significant bit (LSB) Bit 31 (for 32-bit) = most significant bit (MSB) To read bit i of number n: (n >> i) & 1 To read bit 3 of 13: (13 >> 3) & 1 = (1) & 1 = 1 ✓

The Six Bitwise Operators

# a = 0b1100 = 12    b = 0b1010 = 10

a = 12    # 1 1 0 0

a & b     # 1 0 0 0 = 8   — AND:  1 only where BOTH are 1
a | b     # 1 1 1 0 = 14  — OR:   1 where AT LEAST ONE is 1

~a        # -(a+1) = -13  — NOT:  flips all bits; WHY -(a+1): two's complement inversion

a >> 1    # 0 1 1 0 = 6    — right shift: floor divide by 2^n (sign bit replicated on left)

# Shift rules
n << k   ==  n * (2**k)
n >> k   ==  n // (2**k)   # WHY floor: arithmetic right shift replicates sign bit, matching floor div

# ── Check if bit i is set ───
def is_bit_set(n, i):
    # WHY >> before &: shift puts bit i into position 0, then & 1 masks all others
    return (n >> i) & 1 == 1
# Example: is bit 2 set in 12 (=1100)?
(12 >> 2) & 1   # → 1  ✓

# ── Set bit i (turn it ON) ───
def set_bit(n, i):
    # WHY |: OR sets the target bit regardless of its current value; other bits unchanged
    return n | (1 << i)
set_bit(0b1000, 1)  # → 0b1010

# ── Clear bit i (turn it OFF) ───
def clear_bit(n, i):
    # WHY ~(1<0 guard: 0 passes the bitmask test but is not a power of 2
    return n > 0 and (n & (n - 1)) == 0
# A power of 2 has exactly one set bit: 0b1000
# n-1 flips that bit and all below it: 0b0111
# Their AND is 0 iff n has exactly one set bit (i.e., is a power of 2)

# ── Isolate lowest set bit ───
n & (-n)      # e.g., 12 (1100) → 4 (0100)
# WHY -n: two's complement negation inverts all bits then adds 1, which causes
#         a carry ripple that stops exactly at the lowest set bit position,
#         leaving only that bit set in both n and -n.
# Used in Fenwick Tree / BIT for index traversal

# ── Clear lowest set bit ───
n & (n - 1)   # e.g., 12 (1100) → 8 (1000)
# WHY n-1: subtracting 1 borrows from the lowest set bit, clearing it and setting all bits below it;
#          AND with n then clears those lower bits since they are 0 in n

# ── XOR to find unique element ───
# WHY XOR: identical values cancel (a^a=0) and order doesn't matter (XOR is commutative+associative)
# So XOR-ing an array with one unique element leaves only the unique one
nums = [4, 1, 2, 1, 2]
result = 0              # WHY start at 0: 0 is the XOR identity (n^0 = n)
for x in nums:
    result ^= x
# result = 4  (1^1=0, 2^2=0, 4^0=4)

# Swap two integers without a temp variable
a, b = 5, 9
a ^= b   # a = 5^9 = 12   (a now holds a^b)
b ^= a   # b = 9^12 = 5   (b^(a^b) = a — original a extracted)
a ^= b   # a = 12^5 = 9   ((a^b)^a = b — original b extracted)
# Now a=9, b=5  (simpler: just use `a, b = b, a` in Python)
# WHY this works: XOR is its own inverse; applying it twice restores the original value

Bitmask Patterns for Subsets

A bitmask represents a subset: bit i = 1 means element i is included. With n elements there are 2^n subsets, each represented by an integer 0..2^n - 1.

Bitmask subset enumeration — n=3 elements [A, B, C] → 2³=8 subsets (masks 0–7)
maskbit2 (A)bit1 (B)bit0 (C)elements in subset
0b000 = 0000∅ (empty)
0b001 = 1001{C}
0b010 = 2010{B}
0b011 = 3011{B, C}
0b100 = 4100{A}
0b101 = 5101{A, C}
0b110 = 6110{A, B}
0b111 = 7111{A, B, C} (full)
Bit i = 1 means element i is in the subset. Check: mask & (1 << i). Add element i: mask | (1 << i). All subsets of a mask: sub = (sub−1) & mask (iterate down to 0).
n = 3   # elements: [a, b, c]
# Subsets:  000=∅  001={c}  010={b}  011={b,c}  100={a}  101={a,c}  110={a,b}  111={a,b,c}

# Enumerate ALL subsets of n elements
for mask in range(1 << n):   # WHY 1< 0:
    # loop invariant: sub is a nonzero subset of mask (all set bits in sub are also set in mask)
    # process sub as a subset of mask
    sub = (sub - 1) & mask   # WHY (sub-1)&mask: sub-1 flips lowest set bit and all below it;
                              # & mask restricts the result back to bits allowed by mask,
                              # skipping non-mask bits and stepping to the next valid subset
# WHY terminate at sub>0 (not sub>=0): the empty subset (0) is a valid subset but
# the update `(0-1)&mask` wraps to mask itself, causing an infinite loop — handle 0 separately if needed

# Total subsets of a mask: O(3^n) over all masks (not O(4^n)!)
# This is because each element is in 4 states: not in mask, in mask but not sub, in both

Arithmetic & Geometric Series

# Sum of 1 + 2 + 3 + ... + n
n * (n + 1) // 2           # WHY //2: the product n*(n+1) always has one even factor, so exact int division
                           # e.g., n=10 → 55

# Sum of a + (a+d) + (a+2d) + ... (arithmetic series, k terms)
# = k * (2a + (k-1)*d) / 2
# = k * (first + last) / 2   ← use this form when you know the last term directly

# Sum of squares: 1² + 2² + ... + n²
n * (n + 1) * (2*n + 1) // 6
# WHY //6: the product of three consecutive-ish terms is always divisible by 6 (provable by induction)

# Sum of cubes: 1³ + 2³ + ... + n³
(n * (n + 1) // 2) ** 2    # equals square of triangular number — nice identity

# Sum of geometric series: a + a*r + a*r² + ... + a*r^(k-1)
# = a * (r^k - 1) / (r - 1)   for r != 1
# WHY r!=1 guard: denominator is 0 when r=1; that case is just a*k

# Number of pairs from n elements: n*(n-1)/2
# WHY: n choices for first element, n-1 for second, divide by 2 since order doesn't matter
n * (n - 1) // 2

GCD, LCM, and Modular Arithmetic

import math

# GCD — Euclidean algorithm: O(log min(a,b))
math.gcd(48, 18)     # 6
math.gcd(0, 5)       # 5  (gcd(0,n) = n by definition — base case of Euclidean algorithm)
math.gcd(12, 8, 6)   # 2  (Python 3.9+ — multiple args)

# GCD of a list
from functools import reduce
reduce(math.gcd, [12, 8, 6, 4])   # 2

# LCM
math.lcm(4, 6)       # 12  (Python 3.9+)
# Manual: lcm(a,b) = a * b // gcd(a,b)
# WHY divide by gcd: cancels the shared factor counted twice in a*b
def lcm(a, b): return a * b // math.gcd(a, b)

# LCM of a list
reduce(math.lcm, [4, 6, 10])   # 60

# ── Modular Arithmetic ── (critical when answers can be huge)
MOD = 10**9 + 7    # standard competitive programming modulus (prime)
# WHY prime: primality is required for modular inverse via Fermat's little theorem

# These identities hold with any prime modulus:
(a + b) % MOD == ((a % MOD) + (b % MOD)) % MOD
(a * b) % MOD == ((a % MOD) * (b % MOD)) % MOD
(a - b) % MOD == ((a % MOD) - (b % MOD) + MOD) % MOD  # +MOD prevents negative
# WHY +MOD before final %: Python % is always non-negative, but (a%MOD - b%MOD) can be negative;
#                          adding MOD shifts the range to [0, 2*MOD) so the final % corrects it

# Division under modulo — use modular inverse (only when MOD is prime)
# a / b ≡ a * b^(MOD-2) (mod MOD)   by Fermat's little theorem
# WHY MOD-2: Fermat's little theorem gives b^(MOD-1) ≡ 1 (mod MOD), so b^(MOD-2) ≡ b^(-1)
def mod_inverse(b, mod=MOD):
    return pow(b, mod - 2, mod)    # Python's built-in 3-arg pow does modular exponentiation in O(log exp)

(a * mod_inverse(b)) % MOD   # ≡ a/b (mod MOD)

# ── Fast Modular Exponentiation ──
# Python's built-in pow(base, exp, mod) does this in O(log exp)
pow(2, 1000000, MOD)      # 2^1000000 mod (10^9+7) — instant!
pow(base, exp, mod)       # ALWAYS use this 3-arg form instead of (base**exp) % mod
# WHY: (base**exp) computes the full integer first (astronomically large), THEN mods it — catastrophic

# Manual binary exponentiation (to understand how it works):
def fast_pow(base, exp, mod):
    result = 1          # WHY start at 1: multiplicative identity (1*x = x)
    base %= mod         # WHY reduce early: keeps base small before squaring loop begins
    while exp > 0:      # loop invariant: answer = result * base^exp (mod mod)
        if exp & 1:     # WHY &1: checks lowest bit of exp to decide if this power of base contributes
            result = result * base % mod
        base = base * base % mod  # square: moves to next power of 2 of the original base
        exp >>= 1       # WHY >>1: discards the processed bit; terminates when all bits consumed
    return result

Combinatorics

import math

# n choose k (binomial coefficient)
math.comb(10, 3)    # 120   = 10! / (3! * 7!)

# n permute k
math.perm(5, 2)     # 20    = 5! / 3!

# With modular arithmetic (for large n, k)
MOD = 10**9 + 7

def precompute_factorials(n, mod):
    fact = [1] * (n + 1)
    for i in range(1, n + 1):
        fact[i] = fact[i-1] * i % mod      # build prefix factorial table iteratively
    inv_fact = [1] * (n + 1)
    inv_fact[n] = pow(fact[n], mod - 2, mod)  # WHY start from n: compute inv of n! via Fermat's theorem,
                                               # then derive smaller inverses by working backwards
    for i in range(n - 1, -1, -1):
        inv_fact[i] = inv_fact[i+1] * (i+1) % mod  # WHY: inv_fact[i] = inv_fact[i+1] * (i+1)
                                                     # because fact[i] * (i+1) = fact[i+1]
    return fact, inv_fact

def comb_mod(n, k, fact, inv_fact, mod=MOD):
    if k < 0 or k > n: return 0    # WHY guard: C(n,k)=0 for invalid k; avoids index-out-of-bounds
    # nCk = n! / (k! * (n-k)!) — use precomputed inverse factorials to avoid division under mod
    return fact[n] * inv_fact[k] % mod * inv_fact[n-k] % mod

# Pascal's triangle — for repeated nCk queries with small n
def pascal(n):
    C = [[0]*(n+1) for _ in range(n+1)]
    for i in range(n+1):
        C[i][0] = 1          # WHY base case C[i][0]=1: choosing 0 elements from i is always 1 way
        for j in range(1, i+1):
            C[i][j] = C[i-1][j-1] + C[i-1][j]  # WHY: Pascal's identity — include or exclude element i
    return C

Number / Digit Formulas

import math

# Number of digits of n (base 10)
def num_digits(n):
    return len(str(n))           # simplest
    # or: math.floor(math.log10(n)) + 1  for n > 0
    # WHY avoid log10 in general: floating-point rounding can give wrong answer near powers of 10

# Extract all digits of n
def get_digits(n):
    digits = []
    n = abs(n)          # WHY abs: % on negative numbers gives non-intuitive results in some contexts
    while n:            # loop invariant: remaining digits of n not yet extracted
        digits.append(n % 10)   # extracts least-significant digit
        n //= 10                 # strips the extracted digit
    return digits[::-1]   # WHY reverse: loop produces digits LSB-first; reverse for MSB-first display
# e.g., get_digits(1234) → [1, 2, 3, 4]

# Digit sum
def digit_sum(n):
    return sum(int(d) for d in str(n))
    # or: sum(get_digits(n))

# Reverse a number
def reverse_num(n):
    sign = -1 if n < 0 else 1
    return sign * int(str(abs(n))[::-1])

# Check if palindrome number (without string conversion)
def is_palindrome_num(n):
    if n < 0 or (n % 10 == 0 and n != 0): return False
    # WHY early return for n%10==0 (and n!=0): a number ending in 0 can't be a palindrome
    # because that would require it to start with 0, which integers can't
    reversed_half = 0
    while n > reversed_half:     # loop terminates when we've processed half the digits
        reversed_half = reversed_half * 10 + n % 10   # grow reversed side by one digit
        n //= 10                                       # shrink original side by one digit
    # WHY n==reversed_half//10 handles odd-length: middle digit doesn't need to match anything
    return n == reversed_half or n == reversed_half // 10

# Integer square root (exact, no float imprecision)
math.isqrt(17)   # 4   — floor(sqrt(17))
math.isqrt(16)   # 4
# WHY isqrt over int(math.sqrt(n)): sqrt() uses floats; for large n, rounding error can give wrong result

# Check if perfect square
def is_perfect_square(n):
    r = math.isqrt(n)
    return r * r == n   # WHY multiply back: isqrt floors, so squaring verifies it's exact

# Number of trailing zeros in n!
def trailing_zeros_factorial(n):
    count = 0
    # WHY count factors of 5: each trailing zero = one factor of 10 = one pair (2,5);
    #     factors of 2 always exceed factors of 5 in n!, so only 5s are the bottleneck
    while n >= 5:        # loop terminates when no more multiples of 5 remain
        n //= 5          # WHY //=5 iteratively: counts multiples of 5, then 25, then 125, etc.
        count += n       # n after division = count of multiples of 5^k in original n
    return count

Heap Index Formulas

# In a heap stored as array (0-indexed):
parent(i)       = (i - 1) // 2   # WHY -1 before //2: root is at 0; left child of root is 1, parent(1)=(1-1)//2=0 ✓
left_child(i)   = 2 * i + 1      # WHY +1: root at 0 means left child of root is 1 (not 0)
right_child(i)  = 2 * i + 2      # WHY +2: right child is always one past the left child

# In a heap stored as array (1-indexed — used in some competitive programming):
parent(i)       = i // 2         # WHY simpler: root at 1 makes parent(2)=1, parent(3)=1 — cleaner math
left_child(i)   = 2 * i          # WHY no +1: root at 1 means left child of root is exactly 2
right_child(i)  = 2 * i + 1

Clamping and Ranges

# Clamp value to [lo, hi]
clamped = max(lo, min(hi, value))   # WHY min first: caps at hi, then max ensures floor at lo
clamped = min(max(value, lo), hi)   # same thing — both forms are correct and equivalent

# Map a value from [in_min, in_max] to [out_min, out_max]
def linear_map(x, in_min, in_max, out_min, out_max):
    # WHY (x-in_min)/(in_max-in_min): normalizes x to [0,1] relative to input range
    # then scale and shift into the output range
    return (x - in_min) * (out_max - out_min) / (in_max - in_min) + out_min

# ── Encode/decode a string as frequency vector ───
def to_freq(s):
    freq = [0] * 26

import math

# Manhattan distance (grid distance, no diagonals)
def manhattan(r1, c1, r2, c2):
    return abs(r1 - r2) + abs(c1 - c2)
    # WHY abs: distance is always non-negative regardless of direction

# Euclidean distance
def euclidean(r1, c1, r2, c2):
    return math.sqrt((r1-r2)**2 + (c1-c2)**2)
    # or: math.hypot(r1-r2, c1-c2)   — numerically stabler for very large values
    # or: math.dist((r1,c1), (r2,c2))  — Python 3.8+, works for arbitrary dimensions

# Chebyshev distance (king moves on a chessboard — max of abs differences)
def chebyshev(r1, c1, r2, c2):
    return max(abs(r1-r2), abs(c1-c2))
    # WHY max: a king moves one step in any direction per turn; the longer axis determines total moves
